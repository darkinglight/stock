# 投资分析框架 — 芒格 · 塔勒布 · 格林布拉特

## 使用说明

```
请按照「投资分析框架 — 芒格 · 塔勒布 · 格林布拉特」对以下公司进行全面分析：
- 目标公司：[公司名称/股票代码]
- 分析模式：三方辩论模式（推荐）/ 单一框架分析
```

## 一、数据准备清单（执行分析前强制核对）

在执行任何分析前，必须先获取并确认以下数据完整。缺任何一项即暂停，补充后再继续。

### 1.1 基础数据
- 当前股价、52周高/低、总市值
- 所属行业、主营业务、控股股东/实控人
- 上市板块（主板/创业板/科创板/北交所/H股）

### 1.2 财务数据
| 数据项 | 来源 | 期限要求 |
|--------|------|---------|
| 利润表（营收、毛利率、营业利润、归母净利润） | 年报+最新季报 | 近3年年报+近4个季度 |
| 资产负债表（货币资金、有息负债、应收账款、存货、固定资产） | 年报+最新季报 | 近2年+最新季度 |
| 现金流量表（经营/投资/筹资现金流） | 年报+最新季报 | 近3年年报+最新季度 |
| 业务分部数据（分产品/分地区收入及毛利率） | 年报 | 近2年 |
| 投资收益明细 | 年报附注 | 近1年 |
| 资产减值明细 | 年报附注 | 近1年 |
| 关联交易明细 | 年报附注+临时公告 | 近1年 |
| 货币资金附注（存款分布、受限资金、财务公司存款） | 年报附注 | 近1年 |

### 1.3 治理数据
- 股权质押比例及明细
- 近1年股东/董监高减持记录
- 实控人简历及控制的其他企业（企查查/天眼查）
- 监管处罚记录、诉讼公告、问询函
- 员工持股/股权激励方案及业绩考核目标

> **数据缺失处理：** 搜索2次仍无法获取某项数据 → 在报告中标注「数据缺失」并用合理假设替代（标注假设依据）

**🔴 全局失败模式编码（强制执行）：**

| 失败场景 | 触发条件 | 一线修复 | 仍失败兜底 |
|---------|---------|---------|-----------|
| 网络搜索失败 | WebSearch/API连续3次超时或返回空 | 切换数据源（akshare→yfinance→webfetch） | 在报告中标注「数据来源受限，基于可获取数据分析」 |
| 财务数据不可用 | akshare/yfinance返回空或报错 | 使用webfetch抓取东方财富/同花顺页面 | 标注「财务数据缺失」+暂停该维度分析 |
| 最新季报未发布 | 搜索2次未找到当季数据 | 使用最近可用季报+标注日期 | 允许跳过季度分析，但必须标注 |
| 8轮辩论token即将超限 | Round 5时已输出>8000字 | 自动切换为精简模式：跳过Round 6-7，直接Round 8裁决 | 在报告中标注「因篇幅限制采用精简辩论模式」 |
| 用户中途修改目标股票 | 分析过程中用户更换股票 | 终止当前分析，执行Step 0重新检测新股票历史报告 | — |
| 治理数据完全缺失 | 搜索3次无任何治理信息 | 使用公开信息（年报高管列表+股权结构）做最低限度评估 | 治理评分标注「数据不足，评分仅供参考」+禁止一票否决 |
| 历史报告文件损坏 | 读取MD文件报错或内容不完整 | 删除损坏文件，按「未找到历史报告」处理 | 提示用户「历史报告损坏，将生成全新报告」 |

## 二、三方角色（调用Skill）

| 角色 | Skill名称 | 核心视角 | 一票否决权 |
|------|-----------|---------|-----------|
| **芒格** | `munger-perspective` | 伟大企业、护城河、ROIC、Lollapalooza效应 | 无 |
| **塔勒布** | `taleb-perspective` | 尾部风险、反脆弱、不对称性、遍历性 | **有**（治理<3.0或极端风险） |
| **格林布拉特** | `joel-greenblatt-perspective` | 魔法公式、特殊机会、机构强制卖出 | 无 |

> **调用方式：** 使用Skill工具分别调用三个Skill，获取各角色的分析视角。每个角色必须基于其Skill的方法论输出分析。
>
> **🔴 Skill调用失败处理（强制分支）：**
>
> | 触发条件 | 一线修复 | 仍失败兜底 |
> |---------|---------|-----------|
> | 单个Skill调用失败 | 使用该角色核心方法论手动模拟（引用Skill中的心智模型） | 在报告中标注「[角色]视角为框架模拟，非Skill直接输出」 |
> | 全部三个Skill调用失败 | 询问用户「是否降级为手动模拟模式？」 | 用户拒绝 → 仅输出数据汇总+估值分析，跳过三方辩论章节 |
> | Skill调用超时(>60s) | 重试1次 | 同「单个失败」处理 |
> | Skill返回内容为空/格式错误 | 检查Skill是否已加载→重新调用1次 | 同「单个失败」处理 |

## 三、分析流程（5步）

### Step 0：历史报告检测与对比（每次分析前强制执行）

在启动任何分析之前，必须先检查工作目录下是否已存在该股票的分析报告MD文件。

**检测规则：**
1. 扫描工作目录（默认为项目根目录）下所有 `*投资分析报告.md` 文件
2. 使用 Glob 工具搜索 `*投资分析报告.md` 或 `*{股票代码}*投资分析报告.md`
3. 匹配逻辑：文件名中同时包含股票代码和/或公司名称关键词即视为匹配
4. 文件命名规范：`{公司名称}_{股票代码}_投资分析报告.md`（如 `山东药玻_600529_投资分析报告.md`）

**如果找到匹配的历史报告：**

执行以下对比流程：

```
1. 读取历史报告全文
2. 提取历史报告中的关键指标：
   - 报告日期（判断时效性）
   - 股价/市值
   - 财务数据（营收、净利润、毛利率、净利率、ROE）
   - 估值数据（PE、PB、股息率）
   - 治理风险评分
   - 三方辩论结论和评级
   - 操作建议（买入区间、目标价、止损价）
3. 用当前获取的最新数据与历史数据逐项对比
4. 生成「变化摘要」：
   - 哪些指标发生了显著变化（>10%变动或性质变化）
   - 哪些判断需要修正
   - 新增了哪些数据/信息
   - 三方观点是否有调整
5. 基于对比结果，决定最终输出策略：
   - 如果变化显著（多个核心指标变动>10%）→ 执行完整分析流程 + 在报告末尾增加「与历史报告对比」章节
   - 如果变化不大（个别指标小幅波动）→ 跳过完整辩论，仅更新数据部分，在报告末尾增加「与历史报告对比」章节
   - 如果无变化 → 提示用户「报告内容无显著变化」，询问是否仍需更新
```

**如果未找到匹配的历史报告：**
- 执行完整分析流程，生成新报告

**「与历史报告对比」章节格式：**

```markdown
## 与历史报告对比（YYYY-MM-DD更新）

### 变化摘要
| 维度 | 历史值 | 当前值 | 变化幅度 | 判断 |
|------|--------|--------|---------|------|
| 股价 | XX元 | XX元 | +X% | |
| 营收（TTM） | XX亿 | XX亿 | +X% | |
| 净利润（TTM） | XX亿 | XX亿 | +X% | |
| 毛利率 | X% | X% | +Xpp | |
| PE(TTM) | X | X | +X% | |
| 评级 | ⭐X | ⭐X | | |

### 核心变化分析
1. [最重要的变化及原因]
2. [次重要的变化及原因]
3. ...

### 观点调整
- 芒格：[调整/不调整]，原因
- 格林布拉特：[调整/不调整]，原因
- 塔勒布：[调整/不调整]，原因

### 操作建议变化
- 历史买入区间：XX-XX → 当前买入区间：XX-XX
- 历史目标价：XX → 当前目标价：XX
- 历史止损价：XX → 当前止损价：XX
```

**对比分析的决策逻辑：**
- 历史报告日期在30天内 → 仅更新数据，不重做辩论
- 历史报告日期在30-90天 → 更新数据+简要辩论（3轮精简版）
- 历史报告日期超过90天 → 执行完整分析流程
- 历史报告有「一票否决」触发 → 重新评估治理数据，确认是否解除

### Step 1：行业分析（优先）

| 维度 | 关键指标 | 输出 |
|------|---------|------|
| 规模增长 | 行业产值、5年CAGR | 行业周期位置 |
| 竞争格局 | CR5/CR10、市场份额 | 公司地位 |
| 供需关系 | 产能利用率、库存、价格 | 供需判断 |
| 政策环境 | 监管、补贴、环保 | 政策风险等级 |
| 技术变革 | 新技术渗透率、颠覆风险 | 技术风险 |

**行业周期判断：** 复苏期→买入龙头 | 繁荣期→持有 | 衰退期→回避 | 底部期→买入幸存者

**行业定位核查清单（强制）：**
1. 该公司是行业龙头/第二梯队/边缘参与者？
2. 过去3年市场份额是上升/稳定/下降？（搜索关键词：「[公司] 市场份额 202X」）
3. 行业天花板还有多高？（渗透率 vs 成熟市场对标）
4. 是否存在颠覆性技术威胁？

> **CHECKPOINT 0 · 数据获取确认（含最新季度）：**
> 使用WebSearch搜索获取实时行情、最新年度财报、**最新季度财报**。季度数据不可遗漏。
> 最新季度数据与年度数据趋势不一致时（如年报下滑但Q1恢复增长），必须在报告中明确标注并分析趋势变化。

### Step 2：治理风险分析（优先，一票否决依据）

**8个维度，统一权重：**

| # | 维度 | 权重 | 扣分触发条件 |
|---|------|------|-------------|
| 1 | 股权质押 | 15% | 质押比例>50%扣完，>30%扣一半 |
| 2 | 减持行为 | 10% | 清仓式减持扣完 |
| 3 | 资金占用 | 15% | 其他应收款关联方欠款>净资产5%扣完 |
| 4 | **现金锁定** | **20%** | 存款占比>30%且利率<市场2/3扣完 |
| 5 | 关联交易 | 15% | 定价不公允、金额>营收10%扣完 |
| 6 | 薪酬异常 | 10% | 薪酬增长>利润增长3倍扣完 |
| 7 | 股权激励 | 5% | 目标过低（<历史均值）扣完 |
| 8 | 诚信记录 | 10% | **实控人刑事犯罪扣完（0分）** |

**评分细则（每项从5分往下扣）：**
| 扣分程度 | 5分→? | 示例 |
|---------|-------|------|
| 轻微 | 4分 | 质押15%、减持少量、银行理财利率偏低 |
| 中等 | 3分 | 质押35%、多笔减持、关联交易金额较大 |
| 严重 | 2分 | 质押60%、清仓减持、财务公司巨额存款 |
| 极度 | 1分 | 质押80%、同时减持+资金占用+关联交易 |
| 犯罪 | 0分 | 实控人刑事犯罪立案 |

**评级：** 4-5分🟢 | 3-4分🟡 | 2-3分🟠 | 1-2分🔴

**一票否决机制（塔勒布专属）：**
- 治理综合评分 < 3.0 → 自动否决
- 实控人涉及刑事犯罪 → 自动否决
- 现金锁定占比>50%且利率<市场50% → 自动否决

**深挖要点（强制执行）：**
1. 查阅年报"货币资金"附注，确认存款分布
2. 搜索"财务公司""金融服务协议"等关联交易
3. 对比存款利率与市场大额存单利率（2%-3%）
4. 企查查/天眼查：实控人名下是否有其他同行业公司
5. 查阅监管处罚记录、诉讼公告
6. **实控人体外资产与同业竞争深挖：**
   - 搜索「[实控人姓名] 体外公司」「[实控人姓名] 投资」「[实控人姓名] 关联企业」
   - 逐一比对实控人名下所有关联企业的经营范围
   - 特别关注上市后新设立的企业，与上市公司经营存在交集的
   - **三层次判断标准：** 法理层面（产品技术路线是否完全不同）→ 商业层面（同一供应链/同一客户群/分流订单/挖技术人员）→ 激励层面（体外利润归实控人个人，上市公司吃剩饭）
   - **影响：** 关联交易维度必须扣分，成长性可能被锁死，估值施加体外同业折价（10-20%）

> **CHECKPOINT 1 · 治理否决判定：**
> 治理评分完成后，检查是否触发一票否决条件。触发则暂停分析，报告否决原因，询问用户是否继续。

### Step 3：变化拆解（强制触发）

触发条件（任一点满足即触发）：
- 营收同比变化 > 10%
- 毛利率同比变化 > 2个百分点
- 净利润同比变化 > 15%
- 投资收益占净利润比例 > 10%
- 经营现金流与净利润方向背离
- ROE同比变化 > 5个百分点
- 存货/应收账款增速 > 营收增速的1.5倍

| 变化类型 | 拆解方法 | 输出 |
|---------|---------|------|
| 营收大幅变化 | 拆分：销量×单价，分产品/分地区 | 哪个品类/地区驱动？量价分别贡献多少？ |
| 毛利率变化 | 拆分：原材料成本+制造费用+产品结构 | 成本驱动还是结构驱动？持续性如何？ |
| 净利润大幅变化 | 拆分：营业利润+营业外收支+所得税 | 一次性vs持续性？ |
| **投资收益>10%** | 必须分类：联营分红（计入）/ 处置股权（一次性）/ 公允价值变动（剔除） | 调整后净利润 |
| 现金流与利润背离 | 拆分：应收变化+存货变化+应付变化 | 盈利质量判断 |
| ROE变化 | 杜邦分解：净利率×周转率×杠杆 | 哪个因子驱动？可持续吗？ |
| 应收/存货膨胀 | 账龄分析+库龄分析+周转天数变化 | 减值风险预判 |

**变化拆解输出格式：**
```
[变化项目]：X% → Y%（变动幅度）
拆解过程：
- 量价拆分：销量贡献X%，单价贡献X%
- 产品拆分：A产品X%，B产品X%
- 持续性判断：一次性/周期性/结构性
调整后净利润（剔除一次性项）：XX亿元
```

### Step 4：三方辩论（8轮，核心环节）

**辩论前准备：** 在Round 1之前，三方各输出一份独立分析笔记（不少于200字），基于Step 1-3的数据和自己的方法论判断。笔记内容包括：核心逻辑、1-3个关键数据支撑、初步星级。

```
Round 1：初始立场陈述（三方分别给出初步判断和星级）
  - 芒格：生意质量判断（护城河/ROIC/管理层品质）+ 初始星级
  - 格林布拉特：价值判断（ROC/Earnings Yield/特殊机会）+ 初始星级
  - 塔勒布：风险判断（尾部风险/反脆弱性/Skin in the game）+ 初始星级

Round 2：芒格质询格林布拉特
  默认质询方向：「你看到的"便宜"会不会是价值陷阱？低PE背后是不是资产质量在恶化？」
  至少1个具体质疑（引用数据）

Round 3：塔勒布质询芒格
  默认质询方向：「你说的"护城河"会不会只是过去10年的幸存者偏差？什么因素能一夜之间摧毁它？」
  至少1个具体质疑（引用数据）

Round 4：格林布拉特质询塔勒布
  默认质询方向：「你的尾部风险是真实存在的还是理论上的？price in了吗？概率×影响=？」
  至少1个具体质疑（引用数据）

Round 5：数据深挖（变化拆解在此执行）
  - 如有变化拆解触发，在此轮执行
  - 无触发则检查：应收账款/存货周转天数、经营现金流质量
  - 任何一方提出「我需要更多数据来验证X」→ 暂停搜索补充
  - 【新增】检查管理层讨论与分析（MD&A）中指引与实际情况的一致性

Round 6：立场调整（根据Round 5数据调整初始判断）
  必须回答：Round 5的数据有没有改变你的核心假设？你调整了什么？为什么？

Round 7：分歧聚焦（锁定1-2个核心分歧点深入辩论）
  - 每个角色必须指出自己与另外两方的最大分歧点
  - 对分歧点进行第二轮交锋（各至少1个回应）
  - 【新增】如果三方在某一维度上高度一致，标记为「共识」并从分歧清单移除

Round 8：综合裁决（形成最终投资决策）
```

## 五、特殊模式识别（经验积累）

| 模式 | 特征 | 识别 | 处理 |
|------|------|------|------|
| **A：承德露露模式**（现金锁定型） | 大量现金存入关联财务公司，利率低于市场 | 货币资金附注+金融服务协议+利率对比 | 现金锁定维度扣完，治理评分大概率<3.0触发否决 |
| **B：南京高科模式**（隐藏资产型） | 持有上市公司股权市值>自身总市值 | 长期股权投资按市值重估 | SOTP分部估值，格林布拉特重点分析 |
| **C：楚天高速模式**（一次性减值型） | 一次性大额减值导致净利润暴跌 | 资产减值损失+营业利润与归母净利润差距 | 计算正常化净利润，Mr.Market时间套利 |
| **D：嘉化能源模式**（实控人犯罪型） | 实控人涉刑事犯罪 | 诚信记录维度搜索+证监会/法院公告 | 诚信记录0分+治理<3.0+塔勒布否决 |
| **E：中国化学模式**（财务公司放贷型） | 通过关联财务公司向产业链放贷，消耗经营现金流 | 经营现金流骤降+财务公司贷款余额激增 | 调整后经营现金流=报表+财务公司净放贷 |
| **F：凌霄泵业/领恒模式**（体外同业型） | 实控人上市后体外新设同行业公司，占据高毛利新品类 | 企查查查关联企业+比对经营范围+互动平台提问 | 治理评分扣分+估值施加10-20%折价 |
| **G：现金牛+零有息负债型**（保守型） | 大量现金+零有息负债+持续高分红 | 资产负债表+现金流量表 | 估值加分，芒格重点关注管理层诚信 |
| **H：营收稳健+利润暴跌型**（费用扩张型） | 营收微增+管理费用/销售费用暴增吞噬利润 | 费用明细+员工人数变化 | 检查是扩张期投入还是费用失控 |
